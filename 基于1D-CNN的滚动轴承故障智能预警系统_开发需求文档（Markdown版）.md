# 一、项目概述

本系统为基于1D-CNN算法的滚动轴承故障智能预警系统，核心目标是通过训练好的1D-CNN模型对滚动轴承振动数据进行分析，实现故障类型识别与简易预警。系统整体功能以核心需求为主，无复杂业务逻辑，在保证功能完整性的同时严格控制开发难度，适配本科毕业设计的考核要求，便于开发与演示。

# 二、核心功能需求

## 2.1 用户模块（基础必做）

1. 登录功能：支持用户名+密码登录，输入框做非空校验；登录成功后生成JWT Token并保存至前端本地存储，未登录状态下无法访问系统核心页面；登录失败给出友好提示（如“用户名或密码错误”）。

2. 注册功能：支持基础注册表单（用户名、密码、确认密码），做密码一致性校验；注册成功后自动跳转至登录页面，无需复杂的用户信息完善步骤。

3. 退出功能：点击退出按钮销毁本地Token，跳转至登录页面，确保用户信息安全。

## 2.2 首页统计模块

1. 数据可视化：基于Echarts实现3类基础图表，分别为故障类型占比饼图、近期预测次数折线图、故障预警数量柱状图；图表数据从Redis缓存或后端接口获取，无需实时刷新，页面加载时渲染一次即可，满足基础统计展示需求。

2. 页面展示：图表布局简洁规整，占首页主要区域，无需复杂样式美化，保证图表清晰可辨即可。

## 2.3 故障预测模块（核心功能）

1. 数据输入：提供轴承振动原始数据导入入口，支持txt/csv格式单文件上传，文件大小限制在10MB以内；无需实现批量导入、多格式兼容等复杂处理逻辑，简化开发难度。

2. 故障预测：调用预训练的1D-CNN模型对导入的数据进行分析，明确返回预测结果（正常/内圈故障/外圈故障/滚动体故障）；预测结果以弹窗+页面文字形式双重展示，并给出简易预警提示（如“检测到外圈故障，建议及时检修”），贴合预警系统核心需求。

3. 结果保存：预测完成后，系统自动将预测记录保存至预测历史模块，无需用户手动操作，确保记录可追溯。

## 2.4 预测历史模块

1. 记录展示：以表格形式展示所有预测记录，表格字段包含：预测时间、数据文件名称、预测结果、预警状态；仅支持记录查看功能，无需实现筛选、删除、导出等复杂操作，降低开发难度。

## 2.5 工具模块

1. 检测日志：展示系统核心运行日志，日志字段包含：操作时间、操作类型（模型调用/数据导入/接口请求）、操作结果；仅展示最新100条日志，无查询、分页等复杂功能，满足基础日志查看需求。

2. AI辅助问答：基于固定问答库实现简易交互，支持轴承故障基础问题查询（如“滚动轴承常见故障类型”“内圈故障产生原因”）；无需对接大模型，输入问题后匹配预设答案并展示即可，简化开发流程。
