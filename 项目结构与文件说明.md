## 二、Java 源码说明（按包）

### 1. 启动类

| 文件 | 作用 |
|------|------|
| `EarlyWarningApplication.java` | Spring Boot 主类，`main` 入口，扫描并启动整个应用。 |

### 2. common 包

| 文件 | 作用 |
|------|------|
| `Result.java` | 统一 API 返回格式：`Result.ok(data)` / `Result.fail(msg)`，对应 `{ "code": 200, "msg": "操作成功", "data": ... }`。 |

### 3. config 包（配置与全局行为）

| 文件 | 作用 |
|------|------|
| `AppProperties.java` | 读取 `application.properties` 中 `app.*` 配置：上传目录、Python 脚本目录、Python 命令等。 |
| `JwtProperties.java` | JWT 配置：密钥、Token 有效期（2 小时）。 |
| `JwtUtil.java` | JWT 工具：根据 userId、username 生成 Token；解析 Token 得到 Claims（如 userId）。 |
| `JwtInterceptor.java` | 请求拦截器：从 Cookie/Header 取 Token，校验 JWT 并查 Redis；通过后把 `userId`、`username` 写入 request 属性；失败则跳转登录或返回 401。 |
| `WebMvcConfig.java` | 注册 JwtInterceptor：对 `/`、`/predict`、`/history`、`/tools`、`/api/**` 做鉴权；放行 `/login`、`/register`、`/auth/**` 等。 |
| `SecurityConfig.java` | Spring Security 配置：所有请求放行（不启用默认登录页）；仅提供 BCrypt 密码编码器 Bean。 |
| `ModelAdvice.java` | 全局 Model 增强：为所有请求注入 `currentUsername`，供模板顶栏显示当前用户名。 |

### 4. entity 包（数据实体）

| 文件 | 作用 |
|------|------|
| `User.java` | 用户：id、username、passwordHash、createdAt。 |
| `PredictionRecord.java` | 预测记录：id、userId、文件名、路径、故障名、负载、故障尺寸、置信度、是否预警、消息、创建时间。 |
| `OperationLog.java` | 操作日志：id、操作时间、操作类型、操作结果。 |

### 5. repository 包（数据库访问）

| 文件 | 作用 |
|------|------|
| `UserRepository.java` | 用户表：按用户名查询、保存新用户。 |
| `PredictionRecordRepository.java` | 预测记录表：按 id 查、按 userId 查列表、插入。 |
| `OperationLogRepository.java` | 操作日志表：插入一条、按时间范围分页查询、计数。 |

### 6. service 包（业务逻辑）

| 文件 | 作用 |
|------|------|
| `AuthService.java` | 登录（校验用户名密码、返回 token 信息）、注册（密码一致性、重复用户名、BCrypt 加密后入库）。 |
| `BearingPredictService.java` | 调用 Python：.mat 用 `predict_single.py`，.txt/.csv 用 `predict_txt.py`；解析 JSON 输出为 Map 返回。 |
| `PredictionRecordService.java` | 预测记录：根据预测结果保存一条记录（含 userId）、按 userId 查列表、按 id 查单条。 |
| `OperationLogService.java` | 写一条操作日志，按时间范围分页查询。 |
| `StatsService.java` | 首页统计：故障类型占比、近期 7 天预测次数、故障预警数量；优先从 Redis 读，无则查库并写缓存；支持预测成功后清理统计缓存。 |
| `FaqService.java` | AI 问答：根据用户问题匹配预设问答库，返回答案文案。 |

### 7. web 包（控制器）

| 文件 | 作用 |
|------|------|
| `AuthController.java` | 登录页 GET、注册页 GET；POST `/auth/login`（写 Cookie、Redis）、POST `/auth/register`、POST `/auth/logout`（清 Cookie、删 Redis）。 |
| `IndexController.java` | 首页 GET `/`，返回 index 模板。 |
| `PredictController.java` | 故障预测页 GET；POST 表单上传文件，调用预测服务，写日志与记录，返回 result 模板；预测成功后清理首页统计缓存。 |
| `PredictApiController.java` | POST `/api/predict`：接收文件，10MB 与格式校验，写日志、调用预测、保存记录，返回统一 Result；预测成功后清理首页统计缓存。 |
| `HistoryController.java` | GET `/history` 列表（按当前用户）。 |
| `ApiController.java` | 提供 API：`/api/stats/charts` 图表数据、`/api/history/list` 历史列表、`/api/tools/logs`（按时间查询+分页）、`/api/tools/faq` 问答。 |
| `ToolsController.java` | GET `/tools` 重定向到日志子页；提供 `/tools/logs`、`/tools/assistant` 两个子页面。 |

## 三、资源文件说明

### application.properties

- 应用名、端口、上传大小（10MB）。
- `app.upload-dir`、`app.bearing-model-dir`、`app.bearing-python-executable`：上传目录、Python 脚本目录、Python 命令。
- `app.jwt.*`：JWT 密钥与过期时间。
- `spring.data.redis.*`：Redis 地址（Token 与统计缓存）。
- `spring.datasource.*`：MySQL 连接信息。

### schema-mysql.sql

- 建表：`user`、`prediction_record`（含 `user_id`）、`operation_log`。
- 若已有 `prediction_record` 无 `user_id`，需执行注释中的 ALTER。

### templates

- **login.html / register.html**：独立登录、注册页，表单提交到 `/auth/login`、`/auth/register`。
- **index.html**：主框架（顶栏、侧栏、页脚）+ 三个 Echarts 图表，数据来自 `/api/stats/charts`。
- **predict.html**：主框架 + 上传表单（txt/csv/mat，10MB）+ 结果区；提交走 `/api/predict`，结果弹窗 + 页面展示。
- **history.html**：主框架 + 表格（预测时间、文件名、预测结果、预警状态）。
- **tools-logs.html**：工具-日志页，支持按时间搜索与分页（每页 20 条）。
- **tools-ai.html**：工具-AI助理页，固定问答库交互。
- **result.html**：表单提交预测后的结果页（主框架 + 结果文案 + 继续预测/历史链接）。
- **fragments/common.html**：顶栏、侧栏、页脚、退出按钮脚本等公共片段，供各页 `th:replace` 引入。（已删除无入口的 detail.html。）

## 四、Python 脚本说明

| 文件 | 作用 |
|------|------|
| `predict_single.py` | 接收 .mat 文件路径，读取 CWRU 振动信号，分帧、标准化、1D-CNN 预测，输出一行 JSON。 |
| `predict_txt.py` | 接收 .txt/.csv 路径，读取一维数值，分帧、标准化、同一模型预测。 |
| `inference_mapping.py` | 与 `training/main.py` 的故障编码定义保持一致，负责推理标签映射；输出 `fault_name` 为中文可读格式（如 `滚动体故障_0.007英寸_1HP`），并保留 `raw_label_name` 供调试。 |

运行依赖：同目录下需有 `final_cwru_model.h5`、`scaler_mean.npy`、`scaler_scale.npy`。
